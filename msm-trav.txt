common
Old API:
Класс для создания реквеста (интерфейс HttpRequestBodyMapper - для запросов POST, класс CarHireRequestUriMapper - для запросов GET). Класс для обработки ответа от провайдера (интерфейс CommonResponseMapper - принимает параметры и результат). Конфиг-класс, который собирает реквест, респонс, хедеры и другие вспомогательные классы. Так же юнит-тесты для всех классов и integrationTest для проверки работы модуля в целом. Изначально работали с DOM-парсером.
Для реквеста в виде XML мы создаем болванку и туда сетаем динамические параметры. Метод возвращает строку.
В респонсе мы получаем результат. Берем оттуда body и начинаем парсить. В основном это XML. Использовали VTDGen для навигации по документу. С AutoPilot выбирали первоначальный путь. Далье пробегали по документы и сетали в необоходимую модель параметры. Основной метод возвращал результаты в читаемом для движка виде.
New API:
По сути мы делаем тоже самое, только используем StAX-парсер. Соотвественно другие конфиг-классы. Для респона мы реализовываем интерфейс HttpResponseParser, где переопределяем метод parseResponse. Использовали парсер от движка ТСМ CloseableXmlReader. Находим нужные узлы и сетаем необоходимые значения.

Для тестов использовали либу import static org.mockito.Mockito.mock;
